Morgan Stanley software engineering roles typically involve developing scalable applications for financial services, with a focus on full-stack or backend technologies. These positions emphasize collaboration in Agile teams, code quality, and innovation in areas like fraud detection, operations tech, or client portals.

Key Responsibilities
Design, develop, and maintain web applications, APIs, and services using languages like Java, Python, Scala, or JavaScript/TypeScript.

Conduct code reviews, debugging, performance tuning, and refactor codebases for efficiency; integrate tools and prepackaged software into banking ecosystems.

Collaborate with stakeholders on roadmaps, Agile ceremonies, and adopting new technologies like cloud services or AI chatbots.

Required Skills
Proficiency in core programming (e.g., Java, Spring, Python, Angular, Node.js) with experience in databases (SQL/NoSQL like Oracle, Sybase), multithreading, and design patterns.

Knowledge of microservices, REST APIs, Git, CI/CD, Linux, and testing frameworks (Jasmine, Cypress).

Strong problem-solving, communication, and Agile/DevOps experience; financial domain knowledge is a plus.

Qualifications
Roles often require a Bachelor's or Master's in Computer Science or related fields, with 3+ years of experience for associate levels (e.g., backend dev with ETL tools or UI frameworks).

Mumbai openings (e.g., Parametric team) focus on Java/Python developers for scalable solutions.
